# 简化五级流水线数据通路 - 设计规范

## 1. 概述

本文档描述了简化五级流水线数据通路的完整设计规范，包括数据通路、控制信号、指令格式和执行流程。

## 2. 数据通路架构

### 2.1 寄存器组

```
寄存器     位宽      初始值
R1        16位      16'h0004
R2        16位      16'h0006
R3        16位      16'h1234
R7        16位      16'h00FF
```

### 2.2 功能单元

#### 2.2.1 ALU (16位)
支持的操作：
- `+` (加法)
- `-` (减法)
- `*` (乘法)
- `/` (除法)
- `&` (按位与)

#### 2.2.2 存储器 (MEM)
- 容量：足以存储所有数据
- 宽度：16位
- 初始内容：见初始值表

#### 2.2.3 写回部件 (WB)
- 将EX/MEM阶段的结果写回寄存器
- 支持来自ALU和MEM的数据源

### 2.3 流水线寄存器

```
寄存器名    位宽    说明
IF/ID      控制信号 指令译码锁存
ID/EX      控制+数据 执行阶段锁存
EX/MEM     结果数据  存储阶段锁存
MEM/WB     最终结果  写回阶段锁存
```

## 3. 指令格式

### 3.1 指令类型编码

```
op_type    编码      功能
NOP        2'b00    空操作（无任何操作）
ALU        2'b01    ALU操作：rd = rs1 op rs2
LOAD       2'b10    LOAD操作：从内存加载
STORE      2'b11    STORE操作：写入内存
```

### 3.2 ALU操作码

```
alu_op     编码      操作
ADD        3'b000   加法（rd = rs1 + rs2）
SUB        3'b001   减法（rd = rs1 - rs2）
MUL        3'b010   乘法（rd = rs1 * rs2）
DIV        3'b011   除法（rd = rs1 / rs2）
AND        3'b100   按位与（rd = rs1 & rs2）
```

## 4. 控制信号定义

### 4.1 主要控制信号

| 信号名 | 位宽 | 说明 |
|-------|------|------|
| valid | 1位 | 指令有效标志 |
| op_type | 2位 | 操作类型 |
| rs1, rs2, rd | 3位各 | 寄存器选择（支持R1-R7） |
| imm | 16位 | 立即数 |
| alu_op | 3位 | ALU操作码 |
| byte_en | 2位 | 字节使能信号 |

### 4.2 转发控制信号

```
转发源	转发目标	转发信号
 EX    ID/EX   ex_fwd_rs1, ex_fwd_rs2
 MEM   ID/EX   mem_fwd_rs1, mem_fwd_rs2
```

## 5. 执行流程

### 5.1 五级流水线阶段

#### IF阶段 (Instruction Fetch)
- 从指令存储器读取指令
- 将指令传递到IF/ID寄存器

#### ID阶段 (Instruction Decode)
- 解码指令得到控制信号
- 从寄存器堆读取操作数
- 应用数据转发
- 将信息传递到ID/EX寄存器

#### EX阶段 (Execute)
- ALU执行计算
- 计算存储器地址（LOAD/STORE）
- 将结果传递到EX/MEM寄存器

#### MEM阶段 (Memory)
- 执行LOAD/STORE操作
- ALU结果直接通过
- 将结果传递到MEM/WB寄存器

#### WB阶段 (Write Back)
- 将结果写回寄存器堆
- 更新寄存器状态

### 5.2 数据转发策略

**转发优先级**：EX转发 > MEM转发 > 无转发

#### 转发条件

1. **EX转发** (当前周期的EX结果用于下一条指令的EX)
   ```
   条件：下一条指令需要当前EX的结果
   转发路径：EX/MEM → ID/EX
   ```

2. **MEM转发** (MEM阶段结果用于当前EX)
   ```
   条件：当前EX指令需要MEM结果
   转发路径：MEM/WB → 当前EX
   ```

#### 转发实现细节

- 对于LOAD指令，转发在第6周期获取到结果后进行
- 对于ALU指令，转发在第2-3周期间进行
- STORE操作在第5周期后执行，不产生转发源

## 6. 冲突检测与解决

### 6.1 结构冲突

**问题**：寄存器堆同时读写冲突

**解决方案**：
- 使用异步读、同步写设计
- 在周期的不同时刻进行读写操作

### 6.2 数据冲突

**类型**：RAW (Read-After-Write)

**解决方案**：
1. 数据转发（优先采用）
2. 流水线阻塞（仅在必要时）

**阻塞条件**：
- LOAD指令与紧接着的依赖指令
- 此时需要插入NOP气泡

### 6.3 控制冲突

**问题**：分支指令导致的流水线效率降低

**解决方案**：
- 本设计不涉及分支，简化处理

## 8. 时序规范

### 8.1 关键路径

```
CLK周期 = max(寄存器输出延迟 + 组合逻辑延迟 + 建立时间)
```

### 8.2 时钟边沿

- 所有锁存都在上升沿触发
- 输入/输出都相对于时钟同步
